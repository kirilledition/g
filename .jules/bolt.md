## 2026-03-27 - [Logistic Regression Fisher Information Matrix Solve Optimization]
**Learning:** In the logistic regression IRLS inner loop, the equations `I * x_cov = S_cov` and `I * x_cross = I_cross` were previously solved using two separate calls to `jnp.linalg.solve(I, ...)`. However, because `I` is a Fisher information matrix (and thus symmetric and positive-definite), these two calls can be combined into a single matrix stack solved via Cholesky decomposition using `jax.scipy.linalg.cho_solve` (wrapped in a `jax.vmap` for batching). This leverages the symmetric PSD structure and simultaneous RHS solution, cutting the compute time for this specific block of the loop in half (~2x speedup) during microbenchmarks.
**Action:** When solving multiple linear systems involving the same coefficient matrix, stack the right-hand sides and solve them simultaneously. Furthermore, if the coefficient matrix is known to be symmetric and positive-definite (like a Fisher information or covariance matrix), use `jax.scipy.linalg.cho_solve` rather than the general `jnp.linalg.solve` to achieve better performance.

## 2024-03-27 - [Linear Regression Memory & Compute Optimization]
**Learning:** Instantiating the full dense N x M genotype residual matrix inside `compute_linear_association_chunk` (where N is samples and M is variants) caused unnecessary host/device memory pressure and slower execution. Since linear regression only needs the residual sum of squares and the covariance with the phenotype, we can use the geometric properties of orthogonal projections to compute these directly from the raw genotype matrix and the covariate projection without ever instantiating the full residual matrix. Specifically, `G_res^T Y_res = G^T Y_res` because `Y_res` is orthogonal to the covariate span.
**Action:** When implementing statistical models in JAX, avoid materializing intermediate dense matrices if only their column-wise norms or inner products with orthogonal vectors are required. Compute these directly using the original matrices and projections to save memory and compute.
