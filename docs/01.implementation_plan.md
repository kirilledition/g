# Eliminate Mid-Compute Host Synchronization in Logistic Regression

The logistic regression dispatch functions `compute_logistic_association_chunk` and `compute_logistic_association_chunk_with_mask` call `jax.device_get()` mid-pipeline to inspect whether any variant needs Firth fallback. This forces the Python interpreter to completely halt, wait for XLA to finish all queued work, transfer data to host, branch in Python, then push data back. This serializes what should be an asynchronous pipeline.

## User Review Required

> [!IMPORTANT]
> **Memory trade-off**: The new approach runs the Firth vmap on *all* variants in the chunk when fallback is needed (vs. the current batched-64 approach). For `chunk_size > 64`, peak Firth memory increases proportionally. This is bounded by the user-configurable `--chunk-size` flag and should be fine for typical sizes (32–512). Please confirm this is acceptable, or if we should add a `--firth-batch-size` cap that loops with `jax.lax.scan`.

> [!WARNING]
> **One redundant Firth init per non-fallback variant**: When the Firth branch runs, the Firth solver initializes (probabilities, information matrix, penalized log-likelihood) for all variants including non-fallback ones. The `while_loop` exits at 0 iterations for those, but the init cost is nonzero. In practice, this is far cheaper than the current host-sync stall.

## Proposed Changes

### Logistic Regression Kernels

#### [MODIFY] [logistic.py](file:///home/kirill/Projects/g/src/g/compute/logistic.py)

**1. Add `skip_firth` flag to single-variant Firth solver (L916–1060)**

Modify `fit_single_variant_firth_logistic_regression` to accept a new `skip_firth: jax.Array` (scalar bool) parameter. When `True`, initial `FirthState.converged` is set to `True` so `while_loop` exits at 0 iterations. Return dummy-but-valid statistics (NaN beta/SE, `error_code=0`, `converged=True`).

**2. Update vmap (L1063–1068)**

Update `compute_firth_association_chunk_variantwise` vmap `in_axes` to map `skip_firth` along axis 0.

**3. Rewrite `compute_logistic_association_chunk` (L1171–1296) — no-mask path**

Replace with a single `@jax.jit` function that:
1. Computes heuristic pre-dispatch mask on device
2. Runs standard logistic on all variants
3. Computes `fallback_mask = heuristic_mask | ~converged | ~valid`
4. Initializes Firth coefficients on device: `jnp.where(heuristic_mask[:, None], heuristic_init, standard_coefficients)`
5. Calls `jax.lax.cond(jnp.any(fallback_mask), firth_branch, no_firth_branch)`:
   - **`firth_branch`**: runs `compute_firth_association_chunk_variantwise` on ALL variants with `skip_firth=~fallback_mask`, then merges via `jnp.where(fallback_mask, firth_result, standard_result)` per field
   - **`no_firth_branch`**: returns `standard_result` unchanged
6. No `jax.device_get` anywhere. No Python `for` loop. No `FirthIndexBatch`.

**4. Rewrite `compute_logistic_association_chunk_with_mask` (L1299–1420) — mask path**

Same structure as above, but using `compute_standard_logistic_association_chunk_with_mask` and `compute_firth_pre_dispatch_mask` for the masked variants.

**5. Remove dead code**

Delete `FirthIndexBatch` (L118–123), `build_firth_padded_index_batches` (L1071–1098), `initialize_mixed_firth_batch_coefficients` (L1101–1135), and `compute_firth_association_chunk_with_mask` (L1138–1168) since they are subsumed by the new unified dispatch.

No changes to [engine.py](file:///home/kirill/Projects/g/src/g/engine.py) or [models.py](file:///home/kirill/Projects/g/src/g/models.py) — the public API (`compute_logistic_association_chunk` and `compute_logistic_association_chunk_with_mask`) keeps the same signature and return type.
